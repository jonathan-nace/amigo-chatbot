9 ## Usage
10 0. Setup:
11 Edit the docker-compose.yml to provide KAFKA_ADVERTISED_HOST_NAME and KAFKA_ZOOKEEPER_CONNECT host ips.
12 This is the IP of the host on which we are going to run the docker containers for Kafka and Zookeeper.
13 
14 ```
15 KAFKA_ADVERTISED_HOST_NAME: 192.168.86.89
16 KAFKA_ZOOKEEPER_CONNECT: 192.168.86.89:2181
17 ```
18 1. Start a cluster in detached mode:
19 ```docker-compose up -d```
20 
21 It will start 2 containers:
22 kafkadocker_kafka_1 - with kafka running at 9092 mapped to 9092 of localhost
23 kafkadocker_zookeeper_1 - with zookeeper running at 2181 mapped to 2181 of localhost
24 
25 To start a cluster with 2 brokers:
26 ```docker-compose scale kafka=2```
27 
28  You can use docker-compose ps to show the running instances.
29  If you want to add more Kafka brokers simply increase the value passed to
30  docker-compose scale kafka=n
31 
32  If you want to customise any Kafka parameters, simply add them as environment variables
33  in docker-compose.yml. For example:
34  * to increase the message.max.bytes parameter add KAFKA_MESSAGE_MAX_BYTES: 2000000 to the environment section.
35  * to turn off automatic topic creation set KAFKA_AUTO_CREATE_TOPICS_ENABLE: 'false'
36 
37 2. To interact with Kafka container use the kafka-shell as:
38 ```$ start-kafka-shell.sh <DOCKER_HOST_IP> <ZK_HOST:ZK_PORT>```
39 Example:
40 ```$ ./start-kafka-shell.sh 192.168.86.89 192.168.86.89:2181```
41 
42 3. Testing:
43 To test your setup, start a shell, create a topic and start a producer:
44 
45 ```
46 $ $KAFKA_HOME/bin/kafka-topics.sh --create --topic user_msg \
47 --partitions 4 --zookeeper $ZK --replication-factor 1
48 $ $KAFKA_HOME/bin/kafka-topics.sh --describe --topic user_msg --zookeeper $ZK
49 $ $KAFKA_HOME/bin/kafka-console-producer.sh --topic=user_msg \
50 --broker-list=`broker-list.sh`
51 ```
52 
53 Start another shell and start a consumer:
54 ```$ $KAFKA_HOME/bin/kafka-console-consumer.sh --topic=topic --zookeeper=$ZK```
55 
56 Type in producer shell and it should be published to the client's shell.
57

61 command-processor-service/ConsumerLoop.java - polls from user_msg topic
